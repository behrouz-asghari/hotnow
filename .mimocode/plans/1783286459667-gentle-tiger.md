# Plan: Redesign Historical Data - Per Source with Metrics

## Problem
Current API returns meaningless aggregate data. User needs:
- Per-source view showing what titles were trending each day
- Metrics (traffic/views/score) for each title
- Persistence tracking (how many days a title appears)
- Meaningful charts per source

## New API Response Structure

```
GET /api/history?days=30&source=google

{
  "source": "google",
  "dates": ["2026-07-01", "2026-07-02", ...],
  "dailyCount": [
    { "date": "2026-07-01", "count": 10 },
    ...
  ],
  "topTitles": [
    {
      "title": "some trend",
      "daysAppeared": 5,
      "firstSeen": "2026-07-01",
      "lastSeen": "2026-07-05",
      "bestScore": 95,
      "bestTraffic": 100000,
      "bestViews": 50000,
      "dailyData": [
        { "date": "2026-07-01", "score": 90, "traffic": 80000, "views": 40000 },
        ...
      ]
    }
  ]
}
```

## Tasks

### 1. Rewrite History API
**File:** `app/api/history/route.ts`

Query params: `source` (required), `days`, `search`

Logic:
1. Query `raw_trend_items` filtered by source + date range
2. Group by `normalized_title`:
   - Count days appeared (`daysAppeared`)
   - Get first/last seen dates
   - Get best score/traffic/views across all appearances
   - Store daily data for each appearance
3. Sort by `daysAppeared` desc (most persistent first)
4. Return per-source structure

### 2. Rewrite HistoricalView Component
**File:** `app/components/HistoricalView.tsx`

Layout:
- **Source tabs** at top (google, wiki, ninisite, etc.)
- **Line chart**: daily item count for selected source
- **Top titles table**:
  - Title | Days Appeared | First Seen | Last Seen | Best Score | Best Traffic | Best Views
  - Visual indicator for persistence (e.g. bar showing days/total)
  - Click to expand and see daily breakdown
- **Search** to filter titles

### 3. Keep save function as-is
The `saveRawTrendItems` function in `app/lib/pipeline/saveHistory.ts` works correctly - it saves items with metrics. No changes needed.
